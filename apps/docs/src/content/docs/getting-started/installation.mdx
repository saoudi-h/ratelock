## Using Multiple Adapters

You can use different adapters for different rate limiters in the same app:

```typescript
import { createFixedWindowLimiter as createLocalLimiter } from '@ratelock/local'
import { createFixedWindowLimiter as createRedisLimiter } from '@ratelock/redis'

// Local limiter for lightweight checks
const localLimiter = await createLocalLimiter({ limit: 100, windowMs: 60_000 })

// Redis limiter for distributed checks
const redisLimiter = await createRedisLimiter({
    url: 'redis://localhost:6379',
    limit: 1000,
    windowMs: 60_000,
})
```

## Requirements

- **Node.js 22+** (all packages)
- **TypeScript 5+** (recommended, not required)
- Redis 6+ (for `@ratelock/redis`)
- PostgreSQL 14+ (for `@ratelock/postgres`)

> **Note:** `@ratelock/local` also works in browsers and Bun. Runtime compatibility is being expanded — see [Runtime Compatibility](/docs/getting-started/runtime) for details.
