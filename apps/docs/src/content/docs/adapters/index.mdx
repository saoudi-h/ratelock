RateLock adapters provide the storage layer for rate limit state. Each adapter implements all 4 strategies and supports the same API.

## Available Adapters

| Adapter | Package | Best For |
|---------|---------|----------|
| [Local](/docs/adapters/local) | `@ratelock/local` | Single-process apps, zero dependencies |
| [Redis](/docs/adapters/redis) | `@ratelock/redis` | Distributed systems, high traffic |
| [PostgreSQL](/docs/adapters/postgres) | `@ratelock/postgres` | Apps already using Postgres |

## Same API, Different Backends

All adapters expose the same interface:

```typescript
// All adapters support these methods
const limiter = await createFixedWindowLimiter({ ... })

await limiter.check('user:123')        // Check single identifier
await limiter.checkBatch(['a', 'b'])   // Check multiple identifiers
await limiter.destroy()                // Clean up resources
```

## Choosing an Adapter

- **Start with Local** if you're running a single process or developing locally
- **Use Redis** if you have multiple servers or need the highest performance
- **Use PostgreSQL** if you already have Postgres and don't want to add Redis

## Using Multiple Adapters

You can mix adapters in the same application:

```typescript
import { createFixedWindowLimiter as createLocal } from '@ratelock/local'
import { createFixedWindowLimiter as createRedis } from '@ratelock/redis'

// Lightweight limiter for health checks
const healthLimiter = await createLocal({ limit: 1000, windowMs: 60_000 })

// Distributed limiter for API endpoints
const apiLimiter = await createRedis({
    url: 'redis://localhost:6379',
    limit: 100,
    windowMs: 60_000,
})
```
